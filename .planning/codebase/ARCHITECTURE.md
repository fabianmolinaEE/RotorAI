# Architecture
<!-- last_mapped_commit: 7bcb983 | mapped: 2026-06-02 -->

## Pattern

Role-gated SPA with a **swappable data service layer**. Built on TanStack Start (SSR React meta-framework) targeting Cloudflare Workers. File-based routing via TanStack Router. The core architectural bet is that `mockDataService` and `supabaseDataService` are interchangeable behind a typed `DataService` interface — the demo runs on mocks, the real build swaps one file.

## Layers

| Layer | Path | Responsibility |
|-------|------|----------------|
| Routes | `src/routes/` | File-based pages; `_app.*` share authenticated layout |
| Document shell | `src/routes/__root.tsx` | QueryClientProvider, ThemeProvider, RoleProvider, 404/error boundaries |
| App shell | `src/routes/_app.tsx` | Sidebar, header, role switcher, theme toggle layout |
| Global context | `src/app/` | RoleContext, ThemeProvider, roleNav (per-role nav config) |
| Data layer | `src/data/` | DataService interface, mockDataService impl, seed fixtures, types, subsystemMap |
| UI primitives | `src/components/ui/` | 40+ shadcn/Radix UI primitives |
| App components | `src/components/` | PageShell, AppSidebar, RoleSwitcher, ThemeToggle |
| Server functions | `src/lib/api/` | createServerFn pattern with Zod validation |
| SSR entry | `src/server.ts`, `src/start.ts` | SSR entry point and error normalization middleware |

## Key Abstractions

- **`DataService`** — Swappable async interface. Components only call this. `mockDataService` fills it now; `supabaseDataService` replaces it later. Defined in `src/data/dataService.ts`.
- **`Role`** union type (`owner | manager | technician | customer`) — drives nav visibility and landing route in `src/app/roleNav.ts` and `src/app/RoleContext.tsx`.
- **`Subsystem`** entity with mesh keys — reserved for Phase 2 3D viewer (`src/data/subsystemMap.ts`).
- **`PageShell`** — layout wrapper component in `src/components/page-shell.tsx`.
- **`createServerFn`** — RPC pattern for server functions in `src/lib/api/`.

## State Management

| Concern | Mechanism |
|---------|-----------|
| Server/async state | TanStack Query |
| Active role | `RoleContext` (localStorage-backed) in `src/app/RoleContext.tsx` |
| Theme (light/dark) | `ThemeProvider` (localStorage-backed) in `src/app/ThemeProvider.tsx` |
| Local UI state | `useState` per component |

## Entry Points

- **SSR:** `src/start.ts` → `src/server.ts`
- **Client:** `src/router.tsx` → `src/routes/__root.tsx`
- **Marketing:** `src/routes/index.tsx` (public landing page at `/`)

## Data Flow

```
Component
  → DataService interface (src/data/dataService.ts)
    → mockDataService (src/data/mockDataService.ts)
      → seed.ts fixtures (src/data/seed.ts)
```

Future:
```
Component
  → DataService interface

```

- `src/routeTree.gen.ts` is auto-generated by TanStack Router (do not edit manually)
